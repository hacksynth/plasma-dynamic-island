# Security Policy

## Supported versions

Security fixes are only planned for the latest code on the default branch and,
when available, the most recent tagged release.

| Version | Supported |
| --- | --- |
| `main` / latest default branch | Yes |
| Older snapshots or forks | No |

## Reporting a vulnerability

Do not publish exploit details in a public issue.

If the repository host offers private vulnerability reporting, use that first.
If it does not, open a minimal issue asking for a private contact path and wait
for maintainer response before disclosing details publicly.

Include as much of the following as you can:

- affected commit, tag, or installed version
- impacted component or file path
- reproduction steps or proof of concept
- expected security boundary and actual behavior
- whether the issue requires local access, user interaction, or a specific
  Plasma / Qt environment

The maintainer will aim to acknowledge reports within 7 days and will work on
fixes for issues that are reproducible and in scope for the project.
